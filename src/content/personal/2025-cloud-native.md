# 深入 Cloud Native

探索 Kubernetes、微服务架构与分布式系统的实践之路。

## 核心方向

进入云原生领域后，重心放在了以下几个方面：

- **Kubernetes** — 容器编排的事实标准，从基础概念到生产级集群管理
- **微服务架构** — 服务拆分、通信模式、分布式事务的权衡
- **可观测性** — Metrics, Logs, Traces 三大支柱的实践
- **GitOps** — 以 Git 为单一事实来源的持续交付模式

## 关键实践

在实际项目中，逐步建立起了对分布式系统的直觉——知道什么时候该拆分，什么时候该合并；知道一致性和可用性之间的取舍不是理论问题，而是每天都要面对的工程决策。

## 思考

云原生不只是技术栈的升级，更是一种思维方式的转变：从 "如何运行代码" 到 "如何让系统自我修复"。
